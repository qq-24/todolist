## 🔴 严重问题（必须修复）

### 🔴-1 sync() 竞态 — 远端覆盖丢失本地编辑
- 触发方式：用户编辑任务时恰好 30 秒轮询 sync 拉取到远端更新
- 用户感受：刚编辑的内容消失，无提示
- 根因：todo_provider.dart:209 — `_todos = remoteData.todos` 在 fetch await 窗口期间无保护，用户修改被无条件覆盖
- 涉及文件：lib/providers/todo_provider.dart

### 🔴-2 resolveConflict push 失败后冲突死循环
- 触发方式：冲突 → 保留本地 → push 失败（网络抖动）→ 过期 SHA 导致再次 409 → 无限循环
- 用户感受：冲突弹窗反复出现，无法解决
- 根因：todo_provider.dart:204-207 — push 失败未重新 fetch 最新 SHA
- 涉及文件：lib/providers/todo_provider.dart

### 🔴-3 notFound 分支 push 失败显示同步成功
- 触发方式：远端无文件，push 失败（网络错误）
- 用户感受：以为同步成功但实际未推送
- 根因：todo_provider.dart:168-175 — 缺少 else 分支处理 push 失败
- 涉及文件：lib/providers/todo_provider.dart

### 🔴-4 _buildTodoFile 时间戳可能是过去的
- 触发方式：首次 sync 且远端无文件，本地数据从旧缓存加载
- 用户感受：另一台设备推送后会覆盖本地（因为本地时间戳是过去的）
- 根因：todo_provider.dart:222 — _buildTodoFile 用 `_localUpdatedAt` 而非 `DateTime.now()`
- 涉及文件：lib/providers/todo_provider.dart

### 🔴-5 rescheduleAll 泄漏已删除任务的 Android 闹钟
- 触发方式：远端同步删除任务后，旧 AlarmManager 闹钟残留，到期仍触发
- 用户感受：已删除的任务仍然弹出提醒
- 根因：reminder_service.dart:68-75 — 只对新列表调度，不取消旧列表差集的 Android 闹钟
- 涉及文件：lib/services/reminder_service.dart, MainActivity.kt（需新增 cancelAllAlarms 原生方法）

### 🔴-6 Windows 稍后提醒 Timer 未追踪
- 触发方式：点稍后提醒 → 删除任务 → 15 分钟后仍弹通知
- 用户感受：已删除任务再次弹出提醒
- 根因：windows_notification_service.dart:47 — Timer 未存入可追踪集合
- 涉及文件：lib/services/windows_notification_service.dart

### 🔴-7 Windows 退出时 sync 无超时
- 触发方式：网络不通时点托盘菜单"退出"
- 用户感受：点了退出没反应，进程挂起
- 根因：windows_tray_service.dart:73 — `await sync()` 无超时保护
- 涉及文件：lib/services/windows_tray_service.dart

### 🔴-8 Windows exit(0) 绕过 Flutter 生命周期
- 触发方式：点托盘菜单"退出"
- 用户感受：表面正常，但 SharedPreferences 异步写入可能丢失
- 根因：windows_tray_service.dart:75 — dart:io exit(0) 不触发 dispose
- 涉及文件：lib/services/windows_tray_service.dart

### 🔴-9 onWindowClose 异常时 _isHiding 永远为 true
- 触发方式：关闭窗口时 windowManager.getPosition() 或 SharedPreferences 抛异常
- 用户感受：窗口不隐藏，之后再点关闭完全无反应，只能杀进程
- 根因：home_screen.dart:95-127 — 无 try/finally 包裹
- 涉及文件：lib/screens/home_screen.dart

### 🔴-10 草稿保存遗漏 4 个提醒字段
- 触发方式：新建任务设置了提醒选项后不保存直接关闭，重新打开
- 用户感受：标题/描述/截止时间恢复了，但提醒开关/提前量/重复/震动全部丢失
- 根因：todo_edit_sheet.dart:70-87 — _saveDraft/_loadDraft 只处理 title/desc/deadline
- 涉及文件：lib/widgets/todo_edit_sheet.dart

### 🔴-11 Android pending 文件并发写入数据损坏
- 触发方式：两个闹钟几乎同时触发，用户分别操作
- 用户感受：某个待办的完成/贪睡状态丢失
- 根因：MainActivity.kt — appendText 无文件锁，多线程并发可能行交错
- 涉及文件：MainActivity.kt

### 🔴-12 ReminderAlert 单例被后触发闹钟覆盖
- 触发方式：闹钟 A 响铃中，闹钟 B 触发
- 用户感受：A 的震动被 B 清除，B 的震动被 A 的关闭按钮停掉，WakeLock 泄漏
- 根因：ReminderAlert.startVibration 开头调 stop() 清除前一个闹钟的所有状态
- 涉及文件：MainActivity.kt

### 🔴-13 fromJson 对错误类型字段抛 TypeError
- 触发方式：JSON 数据中某字段类型不匹配（如 deadline 为 int）
- 用户感受：应用崩溃或所有 todo 加载失败
- 根因：todo.dart — `as String`/`as int?`/`as bool?` 强制转换无 try-catch
- 涉及文件：lib/models/todo.dart
- 备注：如果 JSON 完全由自身 toJson 生成，风险较低

### 🔴-14 DISABLE_KEYGUARD 权限残留
- 触发方式：应用安装时
- 用户感受：无直接感知，但安全扫描会标记过度权限
- 根因：AndroidManifest.xml:12 — disableKeyguard 代码已移除但权限未清理
- 涉及文件：AndroidManifest.xml

### 🔴-15 应用被杀后 AlarmManager 闹钟不触发（国产 ROM）
- 触发方式：用户从最近任务列表划掉应用，等待闹钟到期
- 用户感受：到时间了没有提醒
- 根因：国产 ROM force-stop 会清除 PendingIntent 和 AlarmManager 闹钟。缺少 BOOT_COMPLETED 接收器（开机后重新注册闹钟）和厂商自启动引导
- 涉及文件：MainActivity.kt, AndroidManifest.xml
