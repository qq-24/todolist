# 跨端同步关闭通知 + 浮窗被主界面覆盖修复

## Bug 修复：浮窗被主界面覆盖

### 根因
executeAlarm() 中通知的 fullScreenIntent 和 contentIntent 指向 MainActivity，系统触发 fullScreenIntent 时启动 MainActivity 覆盖 AlarmActivity。

### Task B1: 通知 Intent 改为指向 AlarmActivity
- 涉及文件：MainActivity.kt executeAlarm()
- 修改说明：将 openIntent 从 Intent(context, MainActivity::class.java) 改为 Intent(context, AlarmActivity::class.java)，传入 message/id/vibrationMode/todoId extras，去掉 FLAG_ACTIVITY_CLEAR_TOP
- 验证方式：提醒触发后主界面不再跑到前台

---

## 新功能：跨端同步关闭通知

### 需求
A 端完成任务 -> 推送 GitHub -> B 端 sync 拉取 -> 发现任务被完成 -> 自动关闭 B 端正在显示的提醒通知/浮窗/震动

### Task F1: Android 端新增 dismissNotification MethodChannel 方法
- 涉及文件：MainActivity.kt
- 修改说明：MethodChannel 新增 dismissNotification 方法，接收 todoId 参数：
  1. 遍历 ReminderAdvance 枚举名计算所有可能的通知 ID（用 stableHash），调用 nm.cancel(id) 取消通知栏通知
  2. 调用 stopService(Intent(context, AlarmForegroundService::class.java)) 停止前台服务（会连带关闭 AlarmActivity 浮窗和震动）

### Task F2: Windows 端维护通知引用 + 新增 dismissNotification 方法
- 涉及文件：lib/services/windows_notification_service.dart
- 修改说明：
  1. 新增 Map<String, LocalNotification> _activeNotifications
  2. showReminder 中将通知存入 map（key 为 todoId）
  3. 新增 dismissNotification(String todoId) 方法：从 map 取出通知调用 .close()，同时调用 cancelSnooze(todoId)

### Task F3: ReminderService 新增 dismissActiveNotification
- 涉及文件：lib/services/reminder_service.dart
- 修改说明：新增 Future<void> dismissActiveNotification(String todoId) 方法：
  - Android：通过 MethodChannel 调用 dismissNotification
  - Windows：调用 _winNotifier?.dismissNotification(todoId)

### Task F4: sync() 覆盖时对比新旧数据，关闭已完成任务的通知
- 涉及文件：lib/providers/todo_provider.dart
- 修改说明：sync() 中远端覆盖本地之前（_todos = remoteData.todos 之前），对比旧 _todos 和新 remoteData.todos：
  1. 找出旧数据未完成但新数据已完成的任务 ID 列表
  2. 对每个 ID 调用 _reminder.dismissActiveNotification(id)
  3. 然后再执行覆盖

## 执行顺序

Task B1 -> Task F1 -> Task F2 -> Task F3 -> Task F4
