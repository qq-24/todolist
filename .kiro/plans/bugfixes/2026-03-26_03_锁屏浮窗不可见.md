# 锁屏提醒浮窗不可见修复

## Requirements

### R1: 锁屏时提醒界面可见
- 描述：锁屏状态下任务到期时，提醒界面应显示在锁屏上方（像闹钟那样）
- 验收标准：
  - [ ] 锁屏时任务到期，提醒界面显示在锁屏上方
  - [ ] 提醒界面 UI 与现有 overlay 浮窗一致（深色卡片、标题、完成/稍后提醒按钮）
  - [ ] 解锁状态下提醒行为不变（overlay 浮窗正常显示）
  - [ ] "完成"和"稍后提醒"按钮功能正常

## Technical Design

### 根因
Android 10+ 安全策略不允许 TYPE_APPLICATION_OVERLAY 浮窗显示在安全锁屏（PIN/指纹/图案）上方。FLAG_SHOW_WHEN_LOCKED 对 overlay 窗口类型无效。

### 修复方案
新建 AlarmActivity，将 ReminderOverlay 的 UI 迁移为 Activity 的 setContentView。Activity 可以通过 showWhenLocked/turnScreenOn 合法显示在锁屏上方。

executeAlarm() 中改为：先启动 AlarmActivity（锁屏和解锁都用 Activity），移除 overlay 方案。

### 详细设计

#### 文件：MainActivity.kt（修改）

1. 新增 AlarmActivity 类（在同一文件中）：
   - onCreate: setShowWhenLocked(true), setTurnScreenOn(true), dismissKeyguard
   - setContentView: 将 ReminderOverlay.show() 中的 UI 构建代码迁移过来（深色卡片、标题、按钮），用 Activity 的 setContentView 替代 WindowManager.addView
   - 窗口样式：透明背景（Theme.Translucent），让卡片浮在锁屏上面
   - "完成"按钮：写 pending_complete.txt → finish()
   - "稍后提醒"按钮：展开选项 → 写 pending_snooze.txt → finish()
   - 关闭按钮：停止震动铃声 → finish()
   - onDestroy: 如果 Service 还在运行，通知 Service 停止

2. 修改 executeAlarm()：
   - 移除 ReminderOverlay.show() 调用
   - 改为 startActivity(AlarmActivity)，传递 message/id/vibrationMode/todoId
   - 保留 fullScreenIntent 通知作为备选

3. 修改 AlarmForegroundService.onDestroy()：
   - 移除 ReminderOverlay.dismiss()
   - 改为 finish AlarmActivity（通过 broadcast 或 static 引用）

4. ReminderOverlay 对象可以保留但不再使用（或直接删除）

#### 文件：AndroidManifest.xml（修改）

1. 注册 AlarmActivity：
   ```xml
   <activity
       android:name=".AlarmActivity"
       android:exported="false"
       android:showWhenLocked="true"
       android:turnScreenOn="true"
       android:theme="@android:style/Theme.Translucent.NoTitleBar"
       android:launchMode="singleTop"
       android:taskAffinity=""
       android:excludeFromRecents="true" />
   ```

### 影响评估
- 解锁状态下：从 overlay 浮窗变为透明 Activity 上的卡片，视觉效果一致
- 锁屏状态下：从不可见变为可见（修复目标）
- 可移除 SYSTEM_ALERT_WINDOW 权限（不再需要 overlay），但保留也无害
