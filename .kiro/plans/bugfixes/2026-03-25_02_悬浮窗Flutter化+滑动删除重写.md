# Bug 修复方案：悬浮弹窗 Flutter 化 + 滑动删除重写

## Requirements

### R1: 悬浮弹窗改用 Flutter 引擎渲染
- 描述：当前原生 View 悬浮窗视觉塑料感强、按钮无 ripple 反馈。改用独立 FlutterEngine 在悬浮窗中渲染 Dart UI，获得完整 MD3 体验。
- 验收标准：
  - [x] 悬浮弹窗使用 Flutter 渲染，有 MD3 主题、ripple 反馈、圆角卡片
  - [x] 按钮点击有视觉反馈（ripple + 颜色变化）
  - [x] 适配系统导航栏（底部安全区域）
  - [x] "稍后提醒"和"知道了"功能正常
  - [x] vibrationMode 参数正确传递到稍后提醒
  - [x] dismiss 时 FlutterEngine 正确销毁，无内存泄漏

### R2: 滑动删除重写为单层 GestureDetector 方案
- 描述：当前 Dismissible + Stack 浮层方案有三个问题：armed 状态下无法右滑取消、确认状态是浮层叠加不统一、整体不够优雅。改用 GestureDetector 自控拖拽 + 单层布局。
- 验收标准：
  - [x] 第一次左滑超阈值松手 → 弹到 armed 位置（露出 80px 删除区域）
  - [x] armed 状态下再次左滑超阈值 → 删除动画 → 真正删除
  - [x] armed 状态下向右滑 → 取消 armed，恢复原位
  - [x] armed 状态下点击红色删除按钮 → 也可删除
  - [x] 点击其他任务项时，已 armed 的项自动恢复
  - [x] 全程单层布局，内容左移自然露出底部删除区域
  - [x] 保留原有的完成动画（涟漪扩散 + 背景渐变）

**lib/overlay_main.dart：**
```dart
@pragma('vm:entry-point')
void overlayMain() {
  WidgetsFlutterBinding.ensureInitialized();
  const channel = MethodChannel('com.mingh.todolist/overlay');

  // 接收原生传来的参数
  channel.setMethodCallHandler((call) async {
    if (call.method == 'setData') {
      // 启动 UI，传入 message/id/vibrationMode
      runApp(OverlayReminderApp(
        message: call.argument('message'),
        id: call.argument('id'),
        vibrationMode: call.argument('vibrationMode'),
        channel: channel,
      ));
    }
  });
}
```

### Task 2: Android 原生层 ReminderOverlay 改写

- 目标：将 ReminderOverlay 从原生 View 改为 FlutterView + 独立 FlutterEngine
- 涉及文件：`android/app/src/main/kotlin/com/mingh/todolist/MainActivity.kt`

**ReminderOverlay 改写：**
```kotlin
object ReminderOverlay {
    private var overlayView: FlutterView? = null
    private var windowManager: WindowManager? = null
    private var flutterEngine: FlutterEngine? = null

    fun show(context: Context, message: String, id: Int, vibrationMode: String = "continuous"): Boolean {
        if (!Settings.canDrawOverlays(context)) return false
        dismiss(context)

        // 1. 创建独立 FlutterEngine，指定 overlayMain 入口
        val engine = FlutterEngine(context)
        engine.dartExecutor.executeDartEntrypoint(
            DartExecutor.DartEntrypoint(
                FlutterInjector.instance().flutterLoader().findAppBundlePath(),
                "overlayMain"
            )
        )
        flutterEngine = engine

        // 2. 注册 overlay MethodChannel
        val channel = MethodChannel(engine.dartExecutor.binaryMessenger, "com.mingh.todolist/overlay")

        // 3. Dart 引擎就绪后发送数据
        channel.invokeMethod("setData", mapOf(
            "message" to message,
            "id" to id,
            "vibrationMode" to vibrationMode
        ))

        // 4. 监听 Dart 回调
        channel.setMethodCallHandler { call, result ->
            when (call.method) {
                "dismiss" -> { ReminderAlert.stop(); dismiss(context); result.success(null) }
                "snooze" -> {
                    ReminderAlert.stop(); dismiss(context)
                    AlarmHelper.set(context, System.currentTimeMillis() + 300000, message, id + 100000, vibrationMode)
                    result.success(null)
                }
                else -> result.notImplemented()
            }
        }

        // 5. 创建 FlutterView 并添加为悬浮窗
        val appContext = context.applicationContext
        val flutterView = FlutterView(appContext)
        flutterView.attachToFlutterEngine(engine)

        val wm = appContext.getSystemService(Context.WINDOW_SERVICE) as WindowManager
        val dm = appContext.resources.displayMetrics
        val params = WindowManager.LayoutParams(
            WindowManager.LayoutParams.MATCH_PARENT,
            dm.heightPixels / 3,  // 给 Flutter 足够渲染空间
            WindowManager.LayoutParams.TYPE_APPLICATION_OVERLAY,
            WindowManager.LayoutParams.FLAG_NOT_FOCUSABLE
                or WindowManager.LayoutParams.FLAG_SHOW_WHEN_LOCKED
                or WindowManager.LayoutParams.FLAG_TURN_SCREEN_ON,
            PixelFormat.TRANSLUCENT
        ).apply { gravity = Gravity.BOTTOM }

        overlayView = flutterView
        windowManager = wm
        wm.addView(flutterView, params)
        return true
    }

    fun dismiss(context: Context) {
        overlayView?.let { v ->
            v.detachFromFlutterEngine()
            windowManager?.let { wm -> try { wm.removeView(v) } catch (_: Exception) {} }
        }
        flutterEngine?.destroy()
        overlayView = null
        windowManager = null
        flutterEngine = null
    }

    // dp 辅助方法可以删除，不再需要
}
```

需要新增 import：
```kotlin

import io.flutter.embedding.engine.FlutterEngine
import io.flutter.embedding.engine.dart.DartExecutor
import io.flutter.embedding.engine.FlutterInjector
```

- 验证方式：构建安装后，测试通知弹出 Flutter 渲染的悬浮窗，按钮可点击
- 测试要点：
  - 正常路径：弹窗显示、稍后提醒、知道了、关闭按钮
  - 边界：FlutterEngine 初始化延迟（1-2秒内弹出即可）
  - 异常：dismiss 后再次 show 不崩溃、连续快速 show/dismiss 不泄漏
  - 资源：dismiss 后 FlutterEngine 正确销毁（logcat 无泄漏警告）

### Task 3: 滑动删除重写

- 目标：用 GestureDetector + AnimationController 替换 Dismissible，实现单层滑动删除 + 二次确认
- 涉及文件：`lib/widgets/todo_item.dart`

**状态机：**
```
[idle] ──左滑超60px松手──→ [armed] ──左滑超160px松手──→ [deleting] → 回调删除
   ↑                         │
   └──右滑回0 / 点其他项──────┘
                             │
                             └──点击删除按钮──→ [deleting] → 回调删除
```

**核心改动：**

移除 Dismissible，改用：
```dart
// 新增状态
late AnimationController _slideController;  // 驱动水平偏移动画
double _dragOffset = 0;   // 实时拖拽偏移
bool _armed = false;

// 常量
static const _armedOffset = -80.0;
static const _armThreshold = -60.0;
static const _deleteThreshold = -160.0;
```

build 方法核心结构：
```dart
GestureDetector(
  onHorizontalDragUpdate: _onDragUpdate,
  onHorizontalDragEnd: _onDragEnd,
  child: ClipRect(
    child: Stack(
      children: [
        // 底层：红色删除区域（始终存在，被内容遮住）
        Positioned.fill(
          child: Container(
            alignment: Alignment.centerRight,
            padding: const EdgeInsets.only(right: 24),
            color: colorScheme.errorContainer,
            child: Column(
              mainAxisSize: MainAxisSize.min,
              children: [
                Icon(Icons.delete_outline, color: colorScheme.onErrorContainer),
                if (_armed) Text('删除', style: TextStyle(color: colorScheme.onErrorContainer, fontSize: 11)),
              ],
            ),
          ),
        ),
        // 上层：内容，通过 AnimatedBuilder 控制偏移
        AnimatedBuilder(
          animation: _slideController,
          builder: (context, child) => Transform.translate(
            offset: Offset(_slideController.value, 0),
            child: child,
          ),
          child: _buildContent(context, colorScheme, todo),
        ),
      ],
    ),
  ),
)
```

手势处理：
```dart
void _onDragUpdate(DragUpdateDetails d) {
  setState(() {
    _dragOffset += d.delta.dx;
    if (!_armed) {
      // 正常状态：只允许左滑（负值），带阻尼
      _dragOffset = _dragOffset.clamp(-200.0, 0.0);
    } else {
      // armed 状态：从 armedOffset 开始，允许双向
      _dragOffset = (_armedOffset + (_dragOffset - _armedOffset))
          .clamp(-300.0, 0.0);
    }
    _slideController.value = _dragOffset;
  });
}

void _onDragEnd(DragEndDetails d) {
  if (!_armed) {
    if (_dragOffset < _armThreshold) {
      _armed = true;
      _animateTo(_armedOffset);
    } else {
      _animateTo(0);
    }
  } else {
    if (_dragOffset < _deleteThreshold) {
      // 飞出屏幕然后删除
      _animateTo(-MediaQuery.of(context).size.width, onComplete: _delete);
    } else if (_dragOffset > _armThreshold / 2) {
      // 右滑取消
      _armed = false;
      _animateTo(0);
    } else {
      _animateTo(_armedOffset);
    }
  }
}
```

_animateTo 用 Tween + AnimationController.animateTo 实现平滑过渡。

**保留原有功能：**
- _rippleController 涟漪动画不变
- _fadeController 完成背景渐变不变
- _buildContent 方法不变
- _onToggle / _onTap 不变

**新增：外部重置 armed 状态**
- 暴露 `resetArmed()` 方法或通过 GlobalKey 调用
- 或者更简单：在 `_onTap` 时通知 parent 重置其他项的 armed 状态（通过回调）

- 验证方式：
  1. 左滑一次 → 弹到 armed 位置，露出红色删除区域
  2. armed 状态下右滑 → 恢复原位
  3. armed 状态下再左滑 → 删除
  4. armed 状态下点击红色区域 → 删除
  5. 完成动画仍正常工作
- 测试要点：
  - 正常路径：左滑 arm → 右滑取消 → 再左滑 arm → 左滑删除
  - 边界：快速连续滑动、滑动中途反向、极小幅度滑动
  - 异常：删除动画中途 widget 被移除
