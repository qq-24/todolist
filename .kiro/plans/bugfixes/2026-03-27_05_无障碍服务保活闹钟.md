# 无障碍服务保活闹钟方案

## 背景
force-stop 会清除所有 PendingIntent、AlarmManager 闹钟和 WorkManager 任务。
无障碍服务是唯一能在 force-stop 后被系统自动重启的机制。

## 修改方案

### Task 1: 新增 AlarmKeepAliveService（无障碍服务）
- 涉及文件：MainActivity.kt
- 新增 AlarmKeepAliveService 类继承 AccessibilityService
- onServiceConnected：调用 AlarmRescheduleHelper.rescheduleFromLocal 重新注册闹钟
- onAccessibilityEvent / onInterrupt：空实现（不监听任何事件）
- 移除 AlarmRescheduleWorker 和 enqueuePeriodicWork（WorkManager 无效）

### Task 2: 无障碍服务配置文件
- 涉及文件：android/app/src/main/res/xml/accessibility_service_config.xml（新建）
- 配置：不监听任何事件类型，description 说明用途

### Task 3: AndroidManifest.xml 注册
- 注册 AlarmKeepAliveService + BIND_ACCESSIBILITY_SERVICE 权限 + meta-data 指向配置文件
- 移除 WorkManager 依赖（build.gradle）

### Task 4: Flutter 侧引导开启无障碍
- 涉及文件：MainActivity.kt（MethodChannel 新增 checkAccessibility/openAccessibilitySettings）
- 涉及文件：lib/main.dart 或 home_screen.dart（启动时检测，未开启则弹引导对话框）
