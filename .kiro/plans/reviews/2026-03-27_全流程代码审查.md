## 🔴 严重问题（必须修复）

### 🔴-1 [同步引擎] _buildTodoFile 用 DateTime.now() 导致 hash 防重复推送失效

- 触发方式：应用空闲时，每 5 秒轮询 sync 都会产生无意义的 push 请求
- 用户感受：加速消耗 GitHub API 配额
- 根因：todo_provider.dart:260 — `_buildTodoFile()` 使用 `DateTime.now()` 而非 `_localUpdatedAt`
- 修复：改用 `_localUpdatedAt` 作为 updatedAt

### 🔴-2 [同步引擎] pending 文件读写跨进程竞态

- 触发方式：用户在悬浮窗点完成的同时，Flutter 端正好执行 processPendingActions
- 用户感受：点了完成但任务没被标记完成
- 根因：todo_provider.dart:82-86 — 先 readAsLinesSync 后 deleteSync，不是原子操作
- 修复：rename 到临时文件再读取

### 🔴-3 [同步引擎] resolveConflict 死循环 + 冲突对话框信息不足

- 触发方式：网络不稳定时选择"保留本地"，push 409 后 re-fetch 也失败
- 用户感受：反复弹冲突对话框，每次都失败；且对话框信息不足无法做明智选择
- 根因：todo_provider.dart:243-249 — re-fetch 失败时 SHA 过期；home_screen.dart:145-166 — 对话框无差异信息
- 修复：re-fetch 失败时清除冲突状态；对话框显示"本地 X 条/远端 Y 条 + 最后修改时间"；失败时不关闭对话框
- 决策：用户选择方案 a

### 🔴-4 [Android 提醒] AlarmForegroundService 多次 onStartCommand 导致多组 Handler 泄漏

- 触发方式：两个闹钟短时间内先后触发
- 用户感受：第一个闹钟的超时 stopSelf 提前终止第二个闹钟
- 根因：MainActivity.kt AlarmForegroundService onStartCommand — handler/checkRunnable/stopSelf 都是局部变量
- 修复：提升为成员变量，onStartCommand 中先 removeCallbacks 旧的再注册新的，onDestroy 中 removeCallbacksAndMessages

### 🔴-5 [Android 提醒] AlarmActivity checkRunnable 在 finish 后泄漏

- 触发方式：任何闹钟弹出 AlarmActivity 后用户操作关闭
- 用户感受：内存泄漏，checkRunnable 持有 Activity 引用阻止 GC
- 根因：MainActivity.kt AlarmActivity onCreate — checkHandler 在 onDestroy 中没有 removeCallbacksAndMessages
- 修复：提升为成员变量，onDestroy 中取消

### 🔴-6 [Android 提醒] Snooze ID (+100000) 可能溢出 Int32 或与正常 ID 碰撞

- 触发方式：多次稍后提醒累加，或原始 ID 接近 Int.MAX_VALUE
- 用户感受：snooze 闹钟覆盖另一个任务的正常闹钟
- 根因：AlarmActivity/ReminderOverlay 中 `id + 100000`；reminder_service.dart:100
- 修复：snooze 使用独立 ID 空间（如对 todoId+"_snooze" 做 hash）

### 🔴-7 [Windows 原生] DefWindowProc 使用已置空的 window_handle_ 而非参数 hwnd

- 触发方式：窗口关闭过程中收到未被 switch 拦截的消息
- 用户感受：可能导致崩溃
- 根因：win32_window.cpp MessageHandler 末尾 `DefWindowProc(window_handle_, ...)` 应为 `DefWindowProc(hwnd, ...)`
- 修复：改用参数 hwnd

### 🔴-8 [Windows 原生] WM_FONTCHANGE 未检查 flutter_controller_ 空指针

- 触发方式：窗口销毁过程中收到 WM_FONTCHANGE 消息
- 用户感受：应用崩溃
- 根因：flutter_window.cpp MessageHandler — 无空指针检查
- 修复：加 `if (flutter_controller_)` 检查

### 🔴-9 [Windows 原生] exit(0) 绕过所有 C++ 清理逻辑

- 触发方式：用户点击托盘菜单"退出"
- 用户感受：Mutex/COM/析构函数全部被跳过
- 根因：windows_tray_service.dart:82 `exit(0)`
- 修复：改为发送 WM_CLOSE 触发正常窗口关闭流程

### 🔴-10 [数据模型] monthly/yearly 重复日期塌陷

- 触发方式：1月31日 monthly → 2月28日 → 3月28日
- 用户感受："每月最后一天"语义丢失
- 根因：todo.dart:42 — nextOccurrence 传入 current.day 而非原始日期
- 修复：Todo 新增 originalDeadlineDay 字段，nextOccurrence 基于原始日期计算
- 决策：用户选择方案 a

### 🔴-11 [主界面] reorder 覆盖已完成任务 sortIndex

- 触发方式：在有已完成任务的列表中拖拽排序
- 用户感受：取消完成后排序位置错乱
- 根因：todo_provider.dart:170-180 — 遍历整个 sortedTodos 更新 sortIndex
- 修复：只更新未完成任务的 sortIndex

### 🔴-12 [主界面] toggleComplete 直接修改 mutable 字段

- 触发方式：直接点击 checkbox 完成任务
- 用户感受：右键菜单路径可能不触发动画
- 根因：todo_provider.dart:155-167 — 直接修改 todo.completed 而非 copyWith
- 修复：统一使用 copyWith

### 🔴-13 [主界面] processPendingActions snooze 直接修改 mutable 字段

- 触发方式：悬浮窗点击"稍后提醒"后回到前台
- 用户感受：deadline 变化可能不触发 UI 刷新
- 根因：todo_provider.dart:117-128
- 修复：统一使用 copyWith

### 🔴-14 [同步引擎] sync() 并发保护使用 SyncStatus 枚举而非独立锁

- 触发方式：_onSyncSuccess 后 _checkPendingSync 和轮询可能同时进入
- 用户感受：极端情况下数据覆盖
- 根因：todo_provider.dart:186 — 守卫条件是 _syncStatus == SyncStatus.syncing
- 修复：独立 bool _syncing 标志，finally 中重置

### 🔴 UX-1+2 [编辑面板] 草稿保存竞态 + 加载覆盖用户输入

- 触发方式：关闭面板后重新打开草稿丢失；打字时草稿加载覆盖已输入内容
- 用户感受：草稿丢失 / 打的字突然消失
- 根因：todo_edit_sheet.dart:48-63
- 修复：dispose 前同步缓存 controller 值；_loadDraft 检查用户是否已输入则跳过
- 决策：用户选择方案 a

### 🔴 UX-3 [删除] 滑动删除和桌面右键删除无确认无撤销

- 触发方式：手滑或误点删除
- 用户感受：任务永久丢失不可恢复
- 根因：todo_item.dart:165, 215, 244
- 修复：桌面端弹确认对话框，移动端 SnackBar 带撤销按钮
- 决策：用户选择方案 b

### 🔴 UX-5 [同步] 错误状态永久锁定无恢复引导

- 触发方式：连续 3 次同步失败
- 用户感受：红叹号一直显示不知道该怎么办
- 修复：tooltip "同步失败，点击重试" + 抽屉显示错误原因 + 错误时降低轮询到 30 秒
- 决策：用户选择方案 b

### 🔴 UX-6+8 [Windows 提醒] 提醒和稍后提醒依赖进程存活

- 触发方式：进程被杀或系统重启后 Timer 丢失
- 用户感受：设了提醒但到时间没响
- 修复：持久化待触发提醒到本地文件，启动时检查并补发过期提醒
- 决策：用户选择方案 a

### 🔴 UX-7 [Windows 窗口] 位置恢复无屏幕边界校验

- 触发方式：外接显示器断开后窗口恢复到屏幕外
- 用户感受：看不到窗口以为应用坏了
- 修复：恢复后检查是否在可见屏幕范围内，超出则重置默认位置
- 文件：main.dart:28-37

## 🟡 体验问题（建议修复，采用推荐方案）

### 同步引擎
- 🟡-1 5 秒轮询消耗 API 配额过快 → 改为智能退避（空闲 30 秒，有修改时 5 秒）
- 🟡-2 sync 期间 await 点可能丢失用户修改 → 覆盖前再次检查 snapshotAt
- 🟡-3 _saveLocal 失败静默吞掉 → 设置 error 状态让 UI 展示警告
- 🟡-4 错误状态自动重置误导用户 → 只在 sync 成功后才清除 error

### Android 提醒
- 🟡-5 dismissNotification 中 stopService 终止所有活跃提醒 → 只停指定 todoId
- 🟡-6 todos.json 轮询读取无线程安全保护 → 使用原子写入
- 🟡-7 FNV-1a 哈希非 ASCII 不一致 → Kotlin 端 `b.toInt() and 0xFF`
- 🟡-8 AlarmReceiver fallback 在 BroadcastReceiver 中执行 → fallback 只发通知

### Android UI
- 🟡-9 AlarmActivity isDismissing 不重置 → 每次新闹钟重置
- 🟡-10 dismissReceiver 广播可靠性 → 使用 LocalBroadcastManager 或 setPackage

### Windows
- 🟡-11 showReminder 覆盖同 todoId 旧通知未 close → 覆盖前先 close()
- 🟡-12 FindWindowW 按标题查找不够可靠 → 加窗口类名参数
- 🟡-13 最大化时不保持 9:16 比例 → 取决于设计意图（保持当前行为）

### 主界面
- 🟡-14 三种完成路径动画不一致 → didUpdateWidget 中也触发沉底动画
- 🟡-15 窗口位置恢复不校验屏幕边界 → 已合并到 🔴 UX-7
- 🟡-16 TodoFile.fromJson todos 字段类型检查不足 → 加类型检查降级空列表

### UX 走查新增
- 🟡 UX-3 保存按钮无防重复点击 → _saving 标志禁用按钮
- 🟡 UX-4 过期时间 SnackBar 被面板关闭吞掉 → 改为保存前确认对话框
- 🟡 UX-5 截止时间显示不含年份 → 非当前年时显示年份
- 🟡 UX-6 重复任务完成缺乏明确反馈 → 短暂 toast "已推进到 X/X"
- 🟡 UX-7 悬浮窗/sync 完成路径缺少沉底动画 → 合并到 🟡-14
- 🟡 UX-8 完成动画期间 checkbox 视觉不同步 → 本地 state 立即显示勾选
- 🟡 UX-9 抽屉内同步状态信息不足 → error 时显示红色"同步失败"
- 🟡 UX-10 冲突对话框无差异详情 → 合并到 🔴-3
- 🟡 UX-11 同步失败无具体错误原因 → 错误类型传递到 UI 层
- 🟡 UX-12 setAlarm 失败静默吞掉 → 返回失败状态提示用户检查权限
- 🟡 UX-13 Windows 稍后提醒固定 15 分钟 → 按钮文案改为"15分钟后提醒"
- 🟡 UX-14 托盘提示文案时态不对 → 改为"关闭窗口后将最小化到托盘"
- 🟡 UX-15 退出时同步等待无视觉反馈 → 显示"正在保存..."
- 🟡 UX-16 通知点击后 todo 已删除时无提示 → SnackBar "该任务已被删除"
- 🟡 UX-17 重复任务通知"完成"按钮文案误导 → 改为"完成本次"

- 🔵-10 _snoozeTimers/_activeNotifications 缺少 dispose → 添加 dispose 方法
- 🔵-11 托盘图标文件不存在时静默失败 → try-catch
- 🔵-12 Todo 类多个字段非 final → 考虑全部改为 final
- 🔵-13 armedNotifier 全局 ValueNotifier 生命周期问题
- 🔵-14 _saveDraft 在 dispose 中调用异步方法 → 已合并到 🔴 UX-1
- 🔵-15 _isDesktop 多文件重复定义 → 提取为全局常量
- 🔵-16 标题无长度限制 → 添加 maxLength 200
- 🔵-17 保存过程无 loading 反馈 → 简单 loading 状态
- 🔵-18 日期时间选择器嵌套 BottomSheet → 可考虑内联展开
- 🔵-19 滑动删除 armed→删除间距较小 → 增大到 -180px
- 🔵-20 armedNotifier 跨页面共享 → 页面级作用域
- 🔵-21 轮询注释与实际不一致 → 更新注释
- 🔵-22 后台轮询未暂停 → paused 时取消 _pollTimer
- 🔵-23 跨端通知消失依赖轮询延迟 → 可接受
- 🔵-24 窗口无最小尺寸限制 → WM_GETMINMAXINFO 设置 ptMinTrackSize
- 🔵-25 最大化时 9:16 约束失效 → ptMaxSize 约束或禁用最大化
