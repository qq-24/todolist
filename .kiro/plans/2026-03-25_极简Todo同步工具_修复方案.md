# 极简 Todo 同步工具 修复方案

## Requirements

### R1: 提醒功能可靠性
- 描述：提醒必须震动循环直到用户交互，铃声有超时保护，悬浮窗无权限时有 fallback，「完成」按钮真正完成任务
- 验收标准：
  - [ ] 震动循环播放直到用户点击按钮或 60 秒超时
  - [ ] 铃声 60 秒后自动停止
  - [ ] 悬浮窗无权限时铃声/震动仍有超时自动停止
  - [ ] 「完成」按钮通过 MethodChannel 标记任务完成
  - [ ] 「稍后提醒」的闹钟可被 Flutter 侧正确取消
  - [ ] 勿扰震动开关在原生侧正确读取

### R2: 数据安全
- 描述：保存操作必须等待持久化完成，冲突解决失败可重试，过期截止时间有提示
- 验收标准：
  - [ ] 编辑面板保存后 await 持久化完成再关闭
  - [ ] 冲突解决 push 失败时保留冲突数据可重试
  - [ ] 设置过去的截止时间时给出警告
  - [ ] _buildTodoFile() 不再每次刷新时间戳

### R3: 代码清理
- 描述：移除未使用依赖，修复存储不一致
- 验收标准：
  - [ ] pubspec.yaml 移除 flutter_local_notifications、android_alarm_manager_plus、timezone
  - [ ] 构建成功且功能正常

## Technical Design

### 修改概览

核心修改集中在三个文件：
1. **MainActivity.kt** — 提醒功能的可靠性修复（震动循环、铃声超时、完成按钮回调、设置读取修复）
2. **todo_provider.dart** — 同步逻辑修复（时间戳不再冗余刷新、冲突解决失败保留状态）
3. **todo_edit_sheet.dart** — 保存 await 持久化、过期时间校验

### 详细设计

**文件：MainActivity.kt**

1. 震动 repeat 参数从 -1 改为 0（循环），添加 60 秒 Handler 超时自动 stop
2. 铃声添加 60 秒超时自动停止
3. ReminderOverlay.show() 返回 Boolean，返回 false 时设置超时自动 stop
4. 「完成」按钮通过 MethodChannel 调用 Flutter 侧 toggleComplete
5. 「稍后提醒」ID 改为通过 MethodChannel 传递给 Flutter 侧记录，以便取消
6. vibrate_in_dnd 改为读取 FlutterSharedPreferences.xml 的 flutter.vibrate_in_dnd key

**文件：todo_provider.dart**

1. _buildTodoFile() 不再更新 _localUpdatedAt，只在 _saveLocal() 中更新
2. resolveConflict() push 失败时不清空 _conflictRemoteData，保留冲突状态
3. 添加 MethodChannel handler 接收原生侧的 toggleComplete 回调

**文件：todo_edit_sheet.dart**

1. _save() 改为 async，await provider.addTodo()/updateTodo() 后再 pop()
2. 保存时校验 _deadline 是否在未来，过去时间弹出 SnackBar 警告

**文件：pubspec.yaml**

1. 移除 flutter_local_notifications、android_alarm_manager_plus、timezone

**文件：reminder_service.dart**

1. cancelReminder 时同时取消 id+100000 的闹钟

## Tasks

### Task 1: 提醒功能可靠性修复
- 优先级：🔴
- 涉及文件：android/app/src/main/kotlin/com/mingh/todolist/MainActivity.kt, lib/services/reminder_service.dart
- 修复内容：问题 1-6, 8
- 代码变更：
  - ReminderAlert.start(): `createWaveform(pattern, 0)` 改为循环 + 60秒 Handler 超时
  - ReminderOverlay.show(): 返回 Boolean，false 时设超时
  - 「完成」按钮: 通过 MethodChannel 回调
  - vibrate_in_dnd: 读取 FlutterSharedPreferences.xml
  - reminder_service.dart _cancelAllForTodo: 同时取消 id+100000
- 验证方式：测试通知 → 震动持续循环 → 点击完成 → 任务标记完成 → 铃声停止
- 测试要点：悬浮窗有权限/无权限两种场景、静音/勿扰/正常三种模式、多个提醒同时触发

### Task 2: 同步逻辑修复
- 优先级：🔴
- 涉及文件：lib/providers/todo_provider.dart
- 修复内容：问题 9, 12
- 代码变更：
  - _buildTodoFile(): 移除 `_localUpdatedAt = DateTime.now()`，改为 `TodoFile(updatedAt: _localUpdatedAt, todos: _todos)`
  - resolveConflict(): push 失败时不清空 _conflictRemoteData/_conflictRemoteSha
- 验证方式：连续修改后检查 GitHub 仓库 commit 数量不应暴增；冲突解决失败后弹窗仍可重试
- 测试要点：30秒轮询不产生冗余推送、冲突解决失败后重试

### Task 3: 编辑面板修复
- 优先级：🔴
- 涉及文件：lib/widgets/todo_edit_sheet.dart
- 修复内容：问题 10, 11
- 代码变更：

  - 保存时检查 `_deadline != null && _deadline!.isBefore(DateTime.now())`，弹 SnackBar 警告
- 验证方式：保存后立即杀 App，重新打开数据仍在；设置过去时间时看到警告
- 测试要点：正常保存、过期时间保存、快速保存后杀 App
