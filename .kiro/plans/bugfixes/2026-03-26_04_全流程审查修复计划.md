## 🔴 严重问题修复方案

### Task 1: sync() 竞态保护 [🔴-1]
- 涉及文件：lib/providers/todo_provider.dart
- 修改说明：sync() 中 fetch 前记录 `_localUpdatedAt` 快照（`final snapshotAt = _localUpdatedAt`），远端覆盖前比对 `_localUpdatedAt != snapshotAt`，若有变化说明 sync 期间用户修改了数据，放弃本次覆盖并标记 _pendingSync 重新同步

### Task 2: resolveConflict push 失败后重新 fetch SHA [🔴-2]
- 涉及文件：lib/providers/todo_provider.dart
- 修改说明：resolveConflict 中 keepLocal push 失败时，重新 fetch 远端获取最新 SHA 更新 `_conflictRemoteSha`，而非直接 return

### Task 3: notFound 分支 push 失败处理 [🔴-3]
- 涉及文件：lib/providers/todo_provider.dart
- 修改说明：notFound 分支中 push 后检查 `pushResult != SyncResult.success` 时调用 `_onSyncError()`

### Task 4: _buildTodoFile 时间戳修复 [🔴-4]
- 涉及文件：lib/providers/todo_provider.dart
- 修改说明：`_buildTodoFile()` 中 `updatedAt` 改用 `DateTime.now()` 而非 `_localUpdatedAt`

### Task 5: rescheduleAll 取消已删除任务的 Android 闹钟 [🔴-5]
- 涉及文件：lib/services/reminder_service.dart, MainActivity.kt
- 修改说明：
  1. MainActivity.kt MethodChannel 新增 `cancelAllAlarms` 方法，遍历已知 todoId 列表取消所有闹钟。或更简单：在 rescheduleAll 中传入旧列表和新列表的 todoId 差集，对差集调用 cancelReminder
  2. reminder_service.dart 维护一个 `_scheduledTodoIds` Set，rescheduleAll 时对比差集取消

### Task 6: Windows 稍后提醒 Timer 追踪 [🔴-6]
- 涉及文件：lib/services/windows_notification_service.dart, lib/services/reminder_service.dart
- 修改说明：WindowsNotificationService 中维护 `Map<String, Timer> _snoozeTimers`，稍后提醒 Timer 存入其中。暴露 `cancelSnooze(todoId)` 方法。ReminderService._cancelAllForTodo 中同时调用 `_winNotifier?.cancelSnooze(todoId)`

### Task 7: Windows 退出 sync 超时 [🔴-7]
- 涉及文件：lib/services/windows_tray_service.dart
- 修改说明：`await _todoProvider.sync().timeout(const Duration(seconds: 5), onTimeout: () {})`

### Task 8: Windows 退出流程改进 [🔴-8]
- 涉及文件：lib/services/windows_tray_service.dart
- 修改说明：exit 前先 `await _todoProvider.sync().timeout(5s)` → `await trayManager.destroy()` → `SystemNavigator.pop()` 替代 `exit(0)`。如果 pop 不生效则 fallback 到 `exit(0)`

### Task 9: onWindowClose try/finally [🔴-9]
- 涉及文件：lib/screens/home_screen.dart
- 修改说明：onWindowClose 方法体用 try/finally 包裹，finally 中 `_isHiding = false`

### Task 10: 草稿保存补全提醒字段 [🔴-10]
- 涉及文件：lib/widgets/todo_edit_sheet.dart
- 修改说明：_saveDraft/_loadDraft/_clearDraft 补充 draft_remind (bool)、draft_reminderAdvances (String, 逗号分隔 name)、draft_vibrationMode (String, name)、draft_repeatMode (String, name)

### Task 11: Android pending 文件并发写入保护 [🔴-11]
- 涉及文件：MainActivity.kt
- 修改说明：pending_complete.txt 和 pending_snooze.txt 的写入用 `synchronized(lock)` 块包裹，lock 为 companion object 中的静态对象

### Task 12: ReminderAlert 多闹钟并发处理 [🔴-12]
- 涉及文件：MainActivity.kt
- 修改说明：ReminderAlert 改为引用计数模式。startVibration 不再调 stop()，而是递增计数器。stop() 递减计数器，计数器归零时才真正停止震动/铃声/释放 WakeLock。每个 AlarmActivity 持有自己的 alertId，stop 时只递减自己的

### Task 13: fromJson 单条容错 [🔴-13]
- 涉及文件：lib/models/todo.dart
- 修改说明：TodoFile.fromJson 中 todos 列表解析改为逐条 try-catch，解析失败的单条 todo 跳过并 debugPrint 警告，不影响其他 todo 加载

### Task 14: 移除 DISABLE_KEYGUARD 权限 [🔴-14]
- 涉及文件：AndroidManifest.xml
- 修改说明：删除 `<uses-permission android:name="android.permission.DISABLE_KEYGUARD"/>`

### Task 15: 应用被杀后闹钟恢复机制 [🔴-15]
- 涉及文件：MainActivity.kt, AndroidManifest.xml, android/app/build.gradle
- 修改说明：
  1. 添加 `RECEIVE_BOOT_COMPLETED` 权限和 BootReceiver，开机后读取本地 todos.json 重新注册所有闹钟
  2. 添加 WorkManager 依赖，注册 PeriodicWorkRequest（每 15 分钟），Worker 中检查是否有即将到期（30 分钟内）的任务，如果有且 AlarmManager 中无对应闹钟则重新注册
  3. WorkManager 的任务即使 force-stop 后也会被系统恢复（这是 WorkManager 的核心特性）
  4. 应用启动时也注册/更新 WorkManager 任务
