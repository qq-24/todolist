# 书房风 · Direction A · Study Room

静态预览文件位于 `index.html`，直接双击或用浏览器打开即可。单文件、零构建、Google Fonts CDN 加载。

## 五张视图（顶部 tab 切换）

1. 要做的 · TASKS — MD3 暗色。左侧 400×820 手机 frame，右侧桌面端三栏（侧栏 / 主列表 / 详情），7 条真实任务，含逾期红色强调、今日、本周、循环（loop 图标）、已完成（删除线）、以及一条从"想做的"升级而来的任务。左下 FAB，侧栏底部有虚线描边的"想做的"入口（带小印章字）。
2. 想做的 · WISHES — 米白宣纸底，杂志刊头 (masthead) + 右侧竖排装饰字。四个分类（想学的 · 想去的 · 想做的 · 想读的）各配一段斜体小序和"肆 / 伍 / 叁"序数。12 格栅格布局，条目大小不等，含一条拉页金句 (pull quote)。悬停浮现两个衬线下划线动作链接"升级成任务 / 做过了"。每条底部斜体"加入 X 天"。底部安静的"过往合集 →"入口。
3. 新建 · ADD — 顶部两块大选择板"这是要做的 / 这是想 做 的"，下方双表单对照。左侧 MD3 任务表单（完整 deadline / reminder / repeat / category pill）。右侧日记纸质感 wish 表单，36px 大字标题输入，一行"无需时间 · 无需提醒 · 无需分量"说明，提交按钮写的是"写进书房"。
4. 过往合集 · COLLECTION — 居中大标题 + "这一年你做了 N 件自己真的想做的事"奖励性文案。2026 / 2025 / 2024 三个 140px 细体年份分块，共 19 条完成条目，每条含"X 月 · 酝酿 Y 天"、一行小反思。零图标，纯排版。
5. 流转 · FLOW — 三章。章壹：长按 wish 升级为 task 的底部 sheet (deadline picker 带 chip)；章贰：逾期任务的温柔提议卡（"这件事，暂时不想做了？放回想做的，哪天再说。"）+ 放回后在书房中的形态；章叁：四组"常见 App 语气 → 书房语气"的措辞对照。

## 设计决策要点

- Noto Serif SC 贯穿两侧标题。Ma Shan Zheng 只作朱红印章字和卷章序数"壹贰叁肆伍"点缀。
- 朱砂红 #B8322F 严格控制为 accent：印章、章节序数、hover 变色、升级动作链接。
- 所有视图底部一行极淡的卷章 footer："书 · 壹 · 卷 —— 要做的"。
- 任务侧色号：MD3 暗色（#141210 / #1c1a17 / ...），primary 用暖金 #e8c280 代替标准紫蓝，与书房调性一致。
- 过渡 300ms ease，hover 揭示动作、微位移、下划线颜色流转。
- 字距：正文 letter-spacing 0.02–0.05em，大写 label 0.3–0.5em。line-height 默认 1.8+，正文段落 1.9–2.0。
- wish grid 用 12 列栅格 + w-3/4/5/6/7/8 变量，条目 xl/lg/md/sm 字号分级模拟杂志排版节奏。
