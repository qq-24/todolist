## Requirements（需求与验收标准）

### R1: 任务 CRUD
- 描述：用户可创建、查看、编辑、删除任务。任务包含标题、描述、截止时间、是否提醒（含提前量）。
- 验收标准：
  - [ ] 点击 FAB 弹出底部面板，填写后保存成功
  - [ ] 点击任务项弹出底部面板，可编辑所有字段
  - [ ] 左滑删除有阻尼感，删除后任务消失
  - [ ] 数据持久化到本地 JSON 文件

### R2: 任务完成
- 描述：勾选任务后播放华丽简约动画，任务变灰沉底。
- 验收标准：
  - [ ] 点击勾选框触发完成动画
  - [ ] 完成的任务变灰并移到列表底部
  - [ ] 可取消完成状态

### R3: 任务排序
- 描述：默认按添加顺序，支持手动拖拽和按截止时间排序切换。
- 验收标准：
  - [ ] 默认按 sortIndex 排序，已完成沉底
  - [ ] 长按拖拽可调整顺序，松手后 sortIndex 更新
  - [ ] 切换到按截止时间排序后再切回，手动顺序不丢失

### R4: GitHub 同步
- 描述：通过 GitHub REST API 同步 todos.json 文件，自动拉取/推送，冲突弹窗选择。
- 验收标准：
  - [ ] 打开 App 自动拉取远端数据
  - [ ] 任何修改后自动推送
  - [ ] 无变化时不推送（hash 比对）
  - [ ] 冲突时弹窗让用户选版本
  - [ ] 同步图标正确反映状态（转圈/打勾/红叹号）
  - [ ] 远端文件不存在时自动创建

### R5: 提醒
- 描述：任务开启提醒后，按选择的提前量在截止时间前触发系统级提醒。
- 验收标准：
  - [ ] Android 端通过系统闹钟触发，App 不在前台也能提醒
  - [ ] Windows 端 App 在前台时弹系统通知
  - [ ] 可选提前量：到期时、15分钟、1小时、1天

### R6: 主题与设置
- 描述：MD3 动态取色，深浅模式可设置，设置项收在侧滑抽屉中。
- 验收标准：
  - [ ] 支持跟随系统 / 强制浅色 / 强制深色
  - [ ] Android 12+ 动态取色生效
  - [ ] 侧滑抽屉包含同步详情、手动同步、排序切换、主题设置

## Technical Design（技术设计）

### 架构概览

```
┌─────────────┐
│  UI Layer   │  home_screen / widgets
├─────────────┤
│  Provider   │  todo_provider / settings_provider
├─────────────┤
│  Services   │  github_sync / local_storage / reminder
├─────────────┤
│  Models     │  todo.dart
└─────────────┘
```

单向数据流：UI → Provider → Service → Provider 通知 UI 刷新。同步操作在 Service 层，Provider 协调本地存储和远端同步。

### 技术栈
- 语言：Dart
- 框架：Flutter 3.x，Material Design 3（useMaterial3: true）
- 目标平台：Android + Windows
- 状态管理：provider
- 核心依赖：http、path_provider、shared_preferences、dynamic_color、flutter_local_notifications、android_alarm_manager_plus、uuid、crypto

### 数据模型

```dart
enum ReminderAdvance { atTime, min15, hour1, day1 }

class Todo {
  final String id;
  String title;
  String description;
  DateTime? deadline;
  bool remind;
  ReminderAdvance reminderAdvance;
  bool completed;
  int sortIndex;
  final DateTime createdAt;
  DateTime updatedAt;

  Map<String, dynamic> toJson();
  factory Todo.fromJson(Map<String, dynamic> json);
}

class TodoFile {
  final int version = 1;
  DateTime updatedAt;
  List<Todo> todos;

  Map<String, dynamic> toJson();
  factory TodoFile.fromJson(Map<String, dynamic> json);
}
```

同步文件格式：
```json
{
  "version": 1,
  "updatedAt": "ISO8601",
  "todos": [...]
}
```

**文件：`lib/config.dart`（新建）**

```dart
const String githubToken = '<YOUR_TOKEN>';

```

**文件：`lib/models/todo.dart`（新建）**
- 职责：Todo 数据模型，JSON 序列化/反序列化
- 包含 Todo 类和 TodoFile 类，字段如上述数据模型

**文件：`lib/services/local_storage_service.dart`（新建）**
- 职责：本地 JSON 文件读写
- 实现思路：用 path_provider 获取应用文档目录，读写 todos.json
- 关键接口：
```dart
class LocalStorageService {
  Future<TodoFile?> load();
  Future<void> save(TodoFile data);
  Future<String> computeHash(TodoFile data); // SHA256
}
```

**文件：`lib/services/github_sync_service.dart`（新建）**
- 职责：GitHub REST API 交互，冲突检测
- 实现思路：
  - Token 和仓库信息从 config.dart 读取
  - GET 拉取时保存 sha，PUT 推送时带 sha 做乐观锁
  - 冲突判断：本地 updatedAt 和远端 updatedAt 都比上次同步时间新 → 冲突
- 关键接口：
```dart
enum SyncResult { success, conflict, noChange, error }

class GithubSyncService {
  Future<(TodoFile?, String?)> fetch();        // 返回 (数据, sha)
  Future<SyncResult> push(TodoFile data, String sha);
  // 409 或 sha 不匹配 → conflict
}
```

**文件：`lib/services/reminder_service.dart`（新建）**
- 职责：跨平台提醒调度
- 实现思路：
  - 统一接口，内部按 Platform.isAndroid / Platform.isWindows 分支
  - Android：android_alarm_manager_plus 设置精确闹钟
  - Windows：flutter_local_notifications + Dart Timer
  - 每次任务变更后重新调度所有活跃提醒
- 关键接口：
```dart
class ReminderService {
  Future<void> scheduleReminder(Todo todo);
  Future<void> cancelReminder(String todoId);
  Future<void> rescheduleAll(List<Todo> todos);
}
```

**文件：`lib/providers/todo_provider.dart`（新建）**
- 职责：核心状态管理，CRUD，协调本地存储和同步
- 实现思路：
  - 持有 List<Todo> 和同步状态（SyncStatus enum: idle/syncing/success/error）
  - 每次 CRUD 操作：更新内存 → 保存本地 → hash 比对 → 有变化则推送远端
  - 拉取逻辑：fetch 远端 → 比对 updatedAt → 无冲突则合并 → 有冲突则暂存等用户选择
  - 提供 sortedTodos getter，根据当前排序方式返回排序后的列表（已完成的始终沉底）
- 关键接口：
```dart
class TodoProvider extends ChangeNotifier {
  List<Todo> get sortedTodos;
  SyncStatus get syncStatus;

  Future<void> addTodo(Todo todo);
  Future<void> updateTodo(Todo todo);
  Future<void> deleteTodo(String id);
  Future<void> toggleComplete(String id);
  Future<void> reorder(int oldIndex, int newIndex);
  Future<void> sync();              // 手动同步
  Future<void> resolveConflict(bool keepLocal);
}
```

**文件：`lib/providers/settings_provider.dart`（新建）**
- 职责：主题模式、排序方式偏好持久化
- 实现思路：shared_preferences 存储
- 关键接口：
```dart
enum SortMode { manual, byDeadline }

class SettingsProvider extends ChangeNotifier {
  ThemeMode themeMode;
  SortMode sortMode;
  Future<void> setThemeMode(ThemeMode mode);
  Future<void> setSortMode(SortMode mode);
}
```

### 同步流程设计

1. 拉取：GET /repos/{owner}/{repo}/contents/todos.json → 拿到文件内容 + sha
2. 比对：远端 updatedAt vs 本地 updatedAt
   - 远端更新 & 本地无修改 → 用远端覆盖本地
   - 本地更新 & 远端无变化 → 推送本地
   - 双方都有更新 → 冲突，弹窗让用户选
3. 推送：PUT /repos/{owner}/{repo}/contents/todos.json，带上 sha 做乐观锁
4. 防重复：推送前算本地数据 SHA256，和上次推送的 hash 比对，相同则跳过

### 影响评估
- 从零建项目，无现有功能受影响
- GitHub API 速率限制：认证用户 5000 次/小时，自用完全够
- Token 硬编码安全风险：仅限自用私有仓库，可接受

### Task 3: GitHub 同步服务
- [x] 目标：实现 GitHub API 交互和冲突检测
- 涉及文件：lib/services/github_sync_service.dart
- 前置依赖：Task 2
- 实现要点：
  1. fetch()：GET 请求，200 → base64 解码 → 解析 JSON → 返回 (TodoFile, sha)；404 → 返回 (null, null)

  3. 冲突检测逻辑：TodoProvider 层比对 updatedAt，不在 sync service 里做
  4. 请求头：Authorization: Bearer $githubToken，Accept: application/vnd.github+json
- 验证方式：手动调用 fetch/push，检查 GitHub 仓库是否出现 todos.json
- 测试要点：正常拉取、正常推送、文件不存在时首次创建、网络错误处理、Token 无效返回 401 的处理

```
  main.dart                  — App 入口，主题配置
  config.dart                — GitHub 配置常量
  models/
    todo.dart                — Todo 数据模型 + JSON 序列化
  services/
    github_sync_service.dart — GitHub API 交互，上传/下载/冲突检测
    local_storage_service.dart — 本地 JSON 文件读写
    reminder_service.dart    — 提醒调度（平台分支）
  providers/
    todo_provider.dart       — 状态管理，CRUD 操作，触发同步
    settings_provider.dart   — 主题/排序偏好
  screens/
    home_screen.dart         — 主界面，任务列表 + FAB + AppBar
  widgets/
    todo_item.dart           — 单个任务项（勾选、滑动删除、动画）
    todo_edit_sheet.dart     — 底部半屏编辑面板
    sync_icon.dart           — 同步状态图标（转圈/打勾/叹号）
    app_drawer.dart          — 侧滑抽屉
```

---

## 预期整体效果

完成后用户体验：
- 打开 App 即看到最新任务列表，无需手动操作
- 添加/编辑任务流畅快速，底部面板一步到位
- 完成任务时有令人愉悦的视觉反馈
- 两端数据通过 GitHub 自动保持同步，偶发冲突有明确的解决方式
- 重要任务到期前收到系统级提醒
- 界面极简干净，MD3 风格统一，不常用功能收在抽屉里不碍眼
